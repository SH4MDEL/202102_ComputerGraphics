Moving Mountain & Maze

모든 명령어 구현하였습니다.
어떤 명령어를 선행하여 입력하지 않으면 작동하지 않는 명령어가 많아 각 명령어의 자세한 기능을 설명드리겠습니다.

o / O : 직각 투영으로 나타냅니다.
p / P : 원근 투영으로 나타냅니다.
x : 카메라가 x축의 양 방향으로 이동합니다. 다시 누르면 정지합니다.
X : 카메라가 x축의 음 방향으로 이동합니다. 다시 누르면 정지합니다.
y : 카메라가 y축의 양 방향으로 이동합니다. 다시 누르면 정지합니다.
Y : 카메라가 Y축의 음 방향으로 이동합니다. 다시 누르면 정지합니다.
x, y 명령어를 이용한 카메라의 이동 시 항상 객체까지 거리가 3.0이 유지되도록 제작했습니다. (동시에 입력해도 유지됩니다. 즉 구 위를 이동합니다.)
그러면서 동시에 카메라의 UP이 y축을 가리키게 하다 보니 명령어 입력시 카메라의 이동이 다소 다이내믹한데,
작동에는 이상이 없습니다.

z : 원근 투영 시 z축의 양의 방향으로 이동합니다.
Z : 원근 투영 시 z축의 음의 방향으로 이동합니다.
말씀드렸다싶이 x, y 명령어는 기본적으로 구 위를 이동하는 상태입니다. 그런데 이 상태에서 z축 이동을 정의하기가 모호합니다. 
저는 카메라가 이동하는 구의 반지름을 증가시키는 방식으로 확대 / 축소 기능을 구현하였습니다.

m / M : 육면체들이 위 아래로 움직이는데, 움직이는 속도와 최대 높이가 전부 랜덤입니다.
r : 미로를 제작하고, 로봇 객체를 소환합니다.
S : 미로에서 객체가 움직입니다. r로 미로를 생성한 상태에서만 작동합니다.
v : 낮은 높이로 변하여 객체의 움직임을 확인하기 쉽게 합니다. m을 눌러 육면체들이 위 아래로 움직이도록 해야 작동합니다.
w, s, a, d : 앞, 뒤, 좌, 우로 이동합니다. 대문자 S를 눌러 일단 객체가 움직이게 해야 작동합니다.
+ : 육면체 이동하는 속도가 증가합니다.
- : 육면체 이동하는 속도가 감소합니다.
(+ / -의 경우 s를 누르지 않은 상태에서는 속도의 변화가 나타나지 않으나 내부적으로 이동 속도가 변합니다.)
1 : 객체 시점으로 나타냅니다. z, x, y 명령어가 전부 작동하지 않습니다.
3 : 초기 쿼터뷰 상태로 나타냅니다.
c : 모든 값 초기화합니다.
q : 종료합니다.

모든 키보드 명령어는 콘솔 창에 출력됩니다.


추가 구현(?)
입력 크기를 5~20이라고 적어두긴 했지만 컴퓨터 사양만 받쳐준다면 맵 제작의 상한이 없도록 구현했습니다.
100 x 100 또는 1000x1000을 입력해도 그에 맞게 미로가 구현되고 미니맵 등이 적절히 수정됩니다.
